## 3. Non-blocking Residuals

### Candles service

`services/candles/service.py` contains defensive handling for malformed input at the tick-processing
level: `except (TypeError, ValueError)` around close-value parsing with `logger.warning` and
`market_state_skipped` counter increment. This provides graceful skip behaviour without crashing.
No dedicated LR-blocking negative-payload test suite is required for LR-closure.

### Signal service

`services/signal/service.py` contains fail-closed handling: `raise ValueError` for missing
`signal_id` (bubbles up to caller as intended), `except json.JSONDecodeError` for malformed stream
messages. This provides crash-prevention for the signal processing path.
No dedicated LR-blocking negative-payload test suite is required for LR-closure.

### Grafana metrics visibility

The `invalid_payloads` counter is exposed via the service stats endpoint. A committed Grafana
screenshot is not a canonical LR-STATE evidence format in this repository; this criterion is
therefore non-blocking for LR-closure.
