## 1. Scope

LR-030 requires shadow-mode execution to keep real execution at zero while still producing auditable runtime evidence.

This document records implemented technical anchors and the committed >24h shadow/soak run evidence for issue #2440.

## 2. Implementation Anchors

- `services/risk/models.py`
  - propagates `decision_contract_v1.input.run_mode` to top-level `run_mode`
- `services/execution/models.py`
  - resolves `run_mode` from top-level payload and nested bundle fallback
- `services/execution/service.py`
  - hard-blocks `run_mode="shadow"` before any executor call
  - increments `execution_shadow_blocked_total`
  - keeps `execution_orders_filled_total` at `0` for shadow-blocked probe orders

## 3. Reproducible Technical Proof

### Unit proof

- `tests/unit/services/test_execution_shadow_gate.py`
  - shadow orders are rejected before executor
  - `shadow_blocked` telemetry increments
  - auditable `REJECTED` result is published
  - nested `decision_contract_v1.input.run_mode` fallback is covered

### Integration proof

- `tests/integration/test_execution_pipeline.py`
  - constructs a Risk-order payload with `decision_contract_v1.input.run_mode="shadow"`
  - proves `Risk -> Execution -> REJECTED`
  - verifies exported metrics contain:
    - `execution_shadow_blocked_total 1`
    - `execution_orders_filled_total 0`

### Workflow proof path

- `.github/workflows/shadow-soak-evidence.yml`
  - runs a deterministic shadow probe against the live stack in CI
  - writes `evidence/shadow_block_probe.json`
  - captures `evidence/endpoints/execution_metrics.txt`
  - generates `evidence/evidence_index.json`
  - evaluates the run via `infrastructure/scripts/soak_gate_eval.py`
  - builds a canonical package manifest via `infrastructure/scripts/build_shadow_evidence_package.py`

## 4. Hard Invariants

The fail-closed workflow gate requires:

- `execution_shadow_blocked_total >= 1`
- `execution_orders_filled_total == 0`
- auditable `REJECTED` `order_result` present in `shadow_block_probe.json`
- `soak_gate_eval.py` exits non-zero if any invariant fails
