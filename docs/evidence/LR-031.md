# LR-031 Evidence: Shadow Metrics Comparison / Gate Evidence

- Issue: `#785`
- Shadow-gate follow-up: `#1128`
- Status: `PARTIAL`
- Last updated: `2026-03-09`

## 1. Scope

LR-031 requires machine-readable shadow metrics that can be evaluated reproducibly.

The current repository state implements the shadow evidence capture and hard gate plumbing needed for audit and replay. It does not yet claim a full baseline-vs-shadow drift report is committed in the repository.

## 2. Implemented Technical Anchors

- `.github/workflows/shadow-soak-evidence.yml`
  - captures post-soak metrics and status endpoints
  - produces deterministic `shadow_block_probe.json`
  - builds a canonical run package under `evidence/packages/<package_id>/`
- `infrastructure/scripts/generate_evidence_index.py`
  - derives machine-readable fields from shadow-soak evidence
- `infrastructure/scripts/soak_gate_eval.py`
  - fail-closed evaluation over indexed metrics and shadow probe artifact
- `infrastructure/scripts/build_shadow_evidence_package.py`
  - validates that a successful run has a complete, checksummed package manifest

## 3. Reproducible Technical Proof

### Evidence index proof

- `tests/test_generate_evidence_index.py`
  - verifies metric extraction from Prometheus text
  - verifies derived fields like `orders_filled`, `shadow_blocked_total`, `source_integrity`
  - verifies fail-closed behavior for missing required evidence sources

### Gate evaluation proof

- `tests/unit/scripts/test_soak_gate_eval.py`
  - passes only when:
    - `shadow_blocked_total >= 1`
    - `orders_filled == 0`
    - `REJECTED` order result exists
    - probe artifact exists
  - fails closed for missing probe, nonzero fills, or absent reject evidence

## 4. Current Evidence Chain

For the shadow-soak workflow, the current machine-readable chain is:

1. `evidence/endpoints/execution_metrics.txt`
2. `evidence/endpoints/risk_metrics.txt`
3. `evidence/shadow_block_probe.json`
4. `evidence/evidence_index.json`
5. `evidence/soak_gate_eval.json`
6. `evidence/package_manifest.json`
7. `evidence/packages/<package_id>/manifest.json`

This is the repo-supported path, within an unpacked successful evidence package, for post-run audit and replay of the shadow gate outcome.

## 5. Current Limit

This document does not claim that a full baseline-vs-shadow comparison report with acceptance thresholds is committed in the repo today.

Because that broader comparison layer is not yet evidenced here, LR-031 remains `PARTIAL` even though the machine-readable shadow gate evidence path is implemented.
