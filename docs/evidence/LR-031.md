## 1. Scope

LR-031 requires machine-readable shadow metrics that can be evaluated reproducibly.

This document covers shadow-prereq evidence only. It does not define the committed
P5 core artifact contract under `reports/p5_canary/<YYYY-MM-DD>/`.

The repository implements the shadow evidence capture, hard gate plumbing, and
baseline-vs-shadow comparison layer. Historical workflow/package evidence has
produced a PASS verdict, but that evidence is shadow-context evidence and is not
equivalent to a committed P5 core artifact set.

## 2. Implemented Technical Anchors

- `.github/workflows/shadow-soak-evidence.yml`
  - captures post-soak metrics and status endpoints
  - produces deterministic `shadow_block_probe.json`
  - builds a canonical run package under `evidence/packages/<package_id>/`
  - records `run_summary.soak_profile` for the collection profile
  - treats `execution_status.mode` as the canonical runtime-mode field for the current shadow-prereq path (`mock`)
- `infrastructure/scripts/generate_evidence_index.py`
  - derives machine-readable fields from shadow-soak evidence
- `infrastructure/scripts/soak_gate_eval.py`
  - fail-closed evaluation over indexed metrics and shadow probe artifact
- `infrastructure/scripts/build_shadow_evidence_package.py`
  - validates that a successful run has a complete, checksummed package manifest

## 3. Reproducible Technical Proof

### Evidence index proof

- `tests/test_generate_evidence_index.py`
  - verifies metric extraction from Prometheus text
  - verifies derived fields like `orders_filled`, `shadow_blocked_total`, `source_integrity`
  - verifies fail-closed behavior for missing required evidence sources

### Gate evaluation proof

- `tests/unit/scripts/test_soak_gate_eval.py`
  - passes only when:
    - `shadow_blocked_total >= 1`
    - `orders_filled == 0`
    - `REJECTED` order result exists
    - probe artifact exists
  - fails closed for missing probe, nonzero fills, or absent reject evidence

package, for post-run audit and replay of the shadow gate and comparison
outcome.

This chain is shadow-prereq evidence. It is not the normative committed P5 core
artifact contract, which is anchored separately under
`reports/p5_canary/<YYYY-MM-DD>/`.

## 5. Comparison Layer

The comparison layer is implemented and calibrated:

- `infrastructure/scripts/shadow_metrics_compare.py`
  - reads `evidence_index.json` and `docs/evidence/lr031_baseline_thresholds.json`
  - produces `shadow_metrics_comparison.json` and `shadow_metrics_comparison.md`
  - verdict: `PASS` | `FAIL` | `UNCALIBRATED` (all-null thresholds → exit 1)
  - fail-closed: unknown spec keys, missing evidence, or uncalibrated → non-zero exit
- `docs/evidence/lr031_baseline_thresholds.json`
  - versioned threshold file; calibrated from lean dry run `23142835184` (2026-03-16)
  - `calibration_status: CALIBRATED`, all 12 thresholds set

The comparison step runs after `Evaluate shadow soak gate` and before
`Build canonical evidence package` in `.github/workflows/shadow-soak-evidence.yml`.
A canonical shadow evidence package requires
`shadow_metrics_comparison verdict == PASS`.
