# LR-042 Evidence: Chaos Drill Network Latency + Packet Loss

- Issue: `#788`
- Control: `LR-042`
- Status: `IMPLEMENTED`
- Last updated: `2026-03-06`

## 1. Scope

LR-042 validates deterministic fail-safe behavior under controlled network degradation:

- `latency_only` (netem delay)
- `packet_loss_only` (netem loss)
- optional `optional_combined` (delay + loss)

The drill extends existing LR-041/LR-040 structures and keeps zero-execution shadow invariants hard.

## 2. Injection Method

Primary runner:

- `scripts/drills/lr042_network_latency_packet_loss_runner.py`

Method:

- injects network faults via `tc netem` on `cdb_risk:eth0`
- uses Docker exec (`-u 0`) and **always** attempts cleanup (`tc qdisc del ... root`)
- no new proxy/mesh/chaos framework is introduced

Support precheck:

- verifies `tc` availability and netem apply/remove capability
- if unsupported, drill writes artifacts with `overall=UNSUPPORTED` and exits with code `3`

## 3. Hard Invariants

Hard FAIL conditions:

- runtime mode not `shadow` (baseline or post-recovery)
- `execution_orders_filled_total` changes from baseline (`LR-030` breach)
- recovery timeout exceeded
- uncontrolled restart spikes
- pipeline stall after recovery (`signals_delta > 0` and `decisions_delta <= 0`)
- LR-031 hard comparison fail (`runtime_mode_shadow`, `zero_real_execution`)

Chaos-window consistency drift is classified as WARN where appropriate (not hidden, not silently ignored).

## 4. Recovery Classification

- `PASS`: recovery `<= 30s`
- `WARN`: recovery `> 30s` and `<= 60s`
- `FAIL`: recovery `> 60s` or no recovery

Recovery requires:

- netem no longer active
- risk/execution status probes healthy again

## 7. Host/Runtime Prerequisites

- Docker daemon reachable
- required containers running (`cdb_redis`, `cdb_postgres`, `cdb_risk`, `cdb_execution`, `cdb_db_writer`)
- target container allows `tc netem` operations (binary + permissions)
- shadow runtime active (`runtime_mode=shadow`)

If prerequisites are missing, the drill fails fast or classifies as `UNSUPPORTED` with explicit reason.
